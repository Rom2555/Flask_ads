# Flask Ads API

REST API для управления объявлениями на Flask + PostgreSQL.

## Возможности
- Полный CRUD (создание, просмотр, обновление, удаление)
- Пагинация списка объявлений
- Единая валидация входных данных (для POST и PATCH)
- Возврат правильных HTTP-статусов (201 Created, 204 No Content и т.д.)

## Запуск проекта

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Настройте переменные окружения:
```bash
cp .env_example .env
# Отредактируйте .env, указав данные для подключения к PostgreSQL
```

3. Инициализируйте базу данных (создание таблиц):
```bash
python init_db.py
```

4. Запустите сервер:
```bash
python app.py
```
Сервер запустится на http://localhost:5000.

## API Endpoints

| Метод | URL | Описание | Успешный статус |
|-------|-----|----------|-----------------|
| `GET` | `/ads` | Получить список объявлений (с пагинацией) | `200 OK` |
| `POST` | `/ads` | Создать новое объявление | `201 Created` |
| `GET` | `/ads/<id>` | Получить объявление по ID | `200 OK` |
| `PATCH` | `/ads/<id>` | Частично обновить объявление | `200 OK` |
| `DELETE` | `/ads/<id>` | Удалить объявление | `204 No Content` |

*Примечание: При ошибках валидации или отсутствии ресурсов API возвращает соответствующий статус (400, 404) и JSON с описанием ошибки.*

## Примеры запросов

### Создать объявление (201)
```bash
curl -X POST http://localhost:5000/ads \
  -H "Content-Type: application/json" \
  -d '{"title": "Продам телефон", "description": "iPhone 15 в идеале", "owner": "Иван"}'
```

### Получить список с пагинацией (200)
```bash
curl "http://localhost:5000/ads?page=1&per_page=10"
```

### Обновить объявление (200)
```bash
curl -X PATCH http://localhost:5000/ads/1 \
  -H "Content-Type: application/json" \
  -d '{"title": "Продам iPhone 15"}'
```

### Удалить объявление (204)
```bash
curl -X DELETE http://localhost:5000/ads/1 -v
```
*(Ключ `-v` в curl покажет, что сервер вернул статус `204 No Content` без тела ответа)*


### Примеры ошибок валидации (400)
Правила валидации единообразно применяются как при создании (`POST`), так и при обновлении (`PATCH`). 

Формат ответа всегда одинаковый (ключ `error`), но текст меняется в зависимости от ошибки:

**1. Ошибка пустого поля:**
```json
{
  "error": "Заголовок не может быть пустым"
}
```

**2. Ошибка превышения длины:**
```json
{
  "error": "Описание превышает допустимую длину (300)"
}
```

**3. Несколько ошибок одновременно:**
Если в запросе допущено несколько ошибок (например, пустой заголовок и слишком длинное имя владельца), API соберет их все в одно сообщение:
```json
{
  "error": "Заголовок не может быть пустым; Имя владельца превышает допустимую длину (100)"
}
```
